### Виконання роботи
* Результати виконання завдань:
    1. **Створено систему управління бібліотекою** з наступними класами:
        - `Book` - клас для представлення книги
        - `Student` - клас для представлення студента
        - `Library` - клас для управління бібліотекою
        - `Stats` - клас для збору статистики читання
    
    2. **Реалізовано enum класи** для типізації станів:
        - `BookStatus` - статус книги (Доступна, Взята, Втрачена)
        - `Action` - тип дії (взяв, повернув)
    
    3. **Клас Book** містить:
        ```python
        class Book:
            def __init__(self, title, author):
                self.title = title
                self.author = author
                self.description = f"|Назва: {self.title};\n|Автор: {self.author};"
                self.status: BookStatus = BookStatus.AVAILABLE
            
            def __str__(self):
                return self.description
        ```
        - Атрибути: назва, автор, опис, статус
        - Метод `__str__` для текстового представлення
    
    4. **Клас Student** реалізує:
        ```python
        class Student:
            def __init__(self, name):
                self.name = name
                self.student_id = id(self)
                self.description = f"|Ім'я: {self.name};\n|ID студента: {self.student_id};"
                self.borrowed_books: list[Book] = []
            
            @property
            def list_borrowed_books(self):
                for b in self.borrowed_books:
                    print(f"Взято книгу: {b}")
        ```
        - Атрибути: ім'я, ID студента, список взятих книг
        - Property декоратор для перегляду взятих книг
    
    5. **Клас Library** керує всією системою:
        - Додавання книг та реєстрація студентів
        - Видача книг студентам (`lend_book`)
        - Повернення книг (`return_book`)
        - Відслідковування доступних книг
        - Property декоратори для отримання списків
    
    6. **Клас Stats** збирає статистику:
        - Відслідковує дати взяття та повернення книг
        - Обчислює кількість днів читання
        - Веде історію всіх операцій
        - Генерує звіти по студентах
        - Працює з датами через `datetime` модуль
    
    7. **Створено UML діаграми** класів та послідовності дій:
        - Діаграма класів показує структуру системи
        - Діаграми послідовності для процесів взяття та повернення книг
        - Використано Mermaid для візуалізації
    
    8. **Реалізовано симуляцію роботи бібліотеки**:
        - Створено 5 книг різних назв
        - Зареєстровано 4 студенти
        - Симуляція роботи впродовж 30 днів
        - Випадковий вибір студентів та книг
        - Збір статистики читання
