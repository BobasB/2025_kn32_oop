# venv - Стандартне віртуальне середовище Python

## Опис

`venv` - це вбудований модуль Python (з версії 3.3), який дозволяє створювати ізольовані віртуальні середовища. Це базовий інструмент для роботи з віртуальними середовищами, який не потребує додаткового встановлення.

## Переваги venv

- ✅ Вбудований в Python (не потрібно встановлювати)
- ✅ Простий у використанні
- ✅ Легкий та швидкий
- ✅ Офіційно підтримується Python Software Foundation

## Недоліки venv

- ❌ Не управляє залежностями автоматично
- ❌ Немає розділення dev/prod залежностей
- ❌ Потрібно вручну оновлювати requirements.txt
- ❌ Немає перевірки конфліктів залежностей

## Основні команди

### Створення віртуального середовища

```bash
python -m venv ./my_env
```

Ця команда створює нову директорію `my_env` з ізольованим Python середовищем.

### Активація середовища

**На macOS/Linux:**
```bash
source my_env/bin/activate
```

**На Windows:**
```bash
my_env\Scripts\activate
```

Після активації в терміналі з'явиться префікс `(my_env)`, що вказує на активне середовище.

### Встановлення пакетів

```bash
pip install jikanpy-v4 Flask
```

### Перегляд встановлених пакетів

```bash
pip list
```

Виведе список всіх встановлених пакетів та їх версій.

### Інформація про конкретний пакет

```bash
pip show requests
```

Показує детальну інформацію про пакет: версію, автора, залежності тощо.

### Збереження залежностей

```bash
pip freeze > requirements.txt
```

Команда зберігає список всіх встановлених пакетів з точними версіями у файл `requirements.txt`.

### Встановлення з файлу залежностей

```bash
pip install -r requirements.txt
```

Встановлює всі пакети, вказані в файлі `requirements.txt`.

### Деактивація середовища

```bash
deactivate
```

Вихід з віртуального середовища та повернення до глобального Python.

## Структура проекту з venv

```
1_venv/
├── my_env/                    # Віртуальне середовище (не комітиться в git)
│   ├── bin/                   # Виконувані файли
│   │   ├── python
│   │   ├── pip
│   │   └── activate
│   ├── lib/                   # Встановлені пакети
│   └── pyvenv.cfg             # Конфігурація середовища
├── requirements.txt           # Основні залежності
└── requirements-dev.txt       # Залежності для розробки
```

## Файли залежностей у проекті

### requirements.txt

Вміщує основні залежності проекту:

```
Flask==3.1.3
jikanpy-v4==1.0.2
aiohttp==3.13.3
requests==2.32.5
# ... та інші залежності
```

### requirements-dev.txt

Вміщує інструменти для розробки:

```
flake8==7.3.0
```

## Практичний приклад

### Створення проекту з нуля

```bash
# Створюємо віртуальне середовище
python -m venv ./my_env

# Активуємо його
source my_env/bin/activate

# Встановлюємо необхідні пакети
pip install jikanpy-v4 Flask

# Зберігаємо залежності
pip freeze > requirements.txt

# Перевіряємо встановлені пакети
pip list
```

### Відтворення середовища на іншій машині

```bash
# Створюємо та активуємо середовище, встановлюємо залежності одночасно
python -m venv ./my_env && source my_env/bin/activate && pip install -r requirements.txt
```

## Робота з бібліотеками для розробки

```bash
# Активуємо середовище
source ./my_env/bin/activate

# Встановлюємо dev-залежності
pip install -r requirements-dev.txt

# Можемо використовувати flake8 для перевірки коду
flake8 ../anime.py
```

## Best Practices

1. **Завжди додавайте my_env/ (або назву вашого venv) до .gitignore**
   ```
   my_env/
   venv/
   env/
   ```

2. **Використовуйте окремі файли для різних типів залежностей:**
   - `requirements.txt` - для продакшн залежностей
   - `requirements-dev.txt` - для інструментів розробки

3. **Регулярно оновлюйте requirements.txt:**
   ```bash
   pip freeze > requirements.txt
   ```

4. **Вказуйте точні версії пакетів** для відтворюваності:
   ```
   Flask==3.1.3  # добре
   Flask         # погано
   ```

5. **Перевіряйте залежності перед комітом:**
   ```bash
   pip list
   pip check  # перевірка на конфлікти
   ```

## Висновок

venv - це простий та надійний інструмент для створення віртуальних середовищ. Він ідеально підходить для невеликих проектів та навчання, але для більших проектів варто розглянути більш просунуті інструменти, такі як Pipenv або Poetry.
